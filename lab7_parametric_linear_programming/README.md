# ЛР 7. Параметрическое линейное программирование

Программа предназначена для выполнения лабораторной работы по теме **«Параметрическое линное программирование»**.

Реализованы требования к программе:

1. собственный алгоритм решения, без `scipy`, `numpy` и других библиотечных оптимизаторов;
2. модуль снимков алгоритма: поэтапный вывод стандартной формы, базисных матриц, обратных матриц, выражений `X_B(lambda)`, оценок небазисных переменных и итоговых интервалов в Markdown-файл;
3. модуль чтения условия из JSON-файла;
4. отдельный модуль запуска.


## Запуск

Из каталога проекта:

```bash
python main.py --input examples/lab7_objective_parameter_example.json --snapshot parametric_lp_snapshot.md
```

Сокращенная форма:

```bash
python main.py -i examples/lab7_objective_parameter_example.json -s parametric_lp_snapshot.md
```

После запуска в консоль выводятся:

- статус решения;
- количество проанализированных базисов;
- количество найденных оптимальных интервалов;
- найденные интервалы параметра;
- контрольное значение параметра внутри каждого интервала;
- значения исходных переменных;
- значение целевой функции в контрольной точке.

Файл `parametric_lp_snapshot.md` содержит полный поэтапный протокол работы алгоритма.

## Формат входного файла

Входной файл имеет формат JSON.

Общий вид:

```json
{
  "parameter_name": "lambda",
  "parameter_interval": [-3, 3],
  "objective": {
    "type": "max",
    "base": [3, 2],
    "parameter": [1, 0]
  },
  "variable_names": ["x1", "x2"],
  "constraints": [
    {
      "coefficients": [1, 1],
      "sign": "<=",
      "rhs": {
        "base": 4,
        "parameter": 0
      }
    }
  ]
}
```

Поля:

| Поле | Описание |
|---|---|
| `parameter_name` | Имя параметра, например `lambda`, `t`, `p` |
| `parameter_interval` | Диапазон изменения параметра `[min, max]`; можно использовать `null` для бесконечной границы |
| `objective.type` | Тип целевой функции: `max` или `min` |
| `objective.base` | Постоянная часть коэффициентов целевой функции |
| `objective.parameter` | Коэффициенты при параметре в целевой функции |
| `variable_names` | Имена исходных переменных |
| `constraints` | Список ограничений |
| `constraints[].coefficients` | Коэффициенты левой части ограничения |
| `constraints[].sign` | Знак ограничения: `<=`, `>=`, `=` |
| `constraints[].rhs.base` | Постоянная часть правой части ограничения |
| `constraints[].rhs.parameter` | Коэффициент при параметре в правой части ограничения |

Целевая функция задается в виде:

```text
z(lambda) = (c0_1 + c1_1 * lambda) * x1 + ... + (c0_n + c1_n * lambda) * xn
```

Правая часть ограничения задается в виде:

```text
b(lambda) = b0 + b1 * lambda
```

## Алгоритм

Программа использует базисный анализ параметрической задачи линейного программирования.

Основные этапы:

1. Задача приводится к стандартной форме.
2. Для ограничений со знаком `<=` добавляются остаточные переменные.
3. Для ограничений со знаком `>=` добавляются избыточные переменные.
4. Перебираются все возможные базисы размера `m`, где `m` — число ограничений.
5. Для каждого невырожденного базиса вычисляется обратная матрица `B^(-1)`.
6. Вычисляется базисное решение:

```text
X_B(lambda) = B^(-1) * b(lambda)
```

7. Из условий `X_B(lambda) >= 0` строится интервал допустимости базиса.
8. Вычисляются оценки небазисных переменных:

```text
r_j(lambda) = c_j(lambda) - c_B(lambda)^T * B^(-1) * A_j
```

9. Для задачи максимизации используется условие оптимальности:

```text
r_j(lambda) <= 0
```

10. Интервал допустимости пересекается с интервалом оптимальности и исходным интервалом параметра.
11. Все подходящие базисы выводятся как интервалы оптимального решения.

## Что записывается в файл снимков

Файл снимков содержит:

1. исходную стандартную форму задачи;
2. матрицу ограничений;
3. для каждого базиса — `B^(-1)`;
4. выражения базисных переменных через параметр;
5. интервал допустимости базиса;
6. оценки небазисных переменных;
7. интервал оптимальности базиса;
8. итоговый интервал базиса;
9. контрольную точку внутри интервала;
10. значения исходных переменных и целевой функции в контрольной точке;
11. итоговую таблицу интервалов.

## Поддерживаемые типы задач

Поддерживаются задачи вида:

```text
max/min z(lambda) = c(lambda)^T x
A x <= b(lambda)
A x >= b(lambda)
A x = b(lambda)
x >= 0
lambda_min <= lambda <= lambda_max
```

Где:

```text
c(lambda) = c0 + c1 * lambda
b(lambda) = b0 + b1 * lambda
```

## Ограничения реализации

Программа рассчитана на учебные задачи небольшого и среднего размера, так как алгоритм перебирает все базисы.

Это удобно для лабораторной работы, потому что в снимках можно увидеть полный анализ базисов и интервалы, на которых каждый базис является оптимальным.

## Как использовать в отчете

В отчет можно включить:

1. постановку задачи;
2. запись параметрической математической модели;
3. стандартную форму задачи;
4. фрагменты файла снимков с анализом базисов;
5. итоговую таблицу интервалов параметра;
6. оптимальные значения переменных на каждом интервале;
7. значение целевой функции в контрольной точке;
8. листинг программы.

## Возможные статусы

| Статус | Значение |
|---|---|
| `optimal_intervals` | Найдены интервалы параметра, на которых существуют оптимальные базисы |
| `no_optimal_interval` | На заданном диапазоне не найдено оптимального базиса |

Статус `no_optimal_interval` может означать, что задача на указанном диапазоне параметра несовместна, неограничена или входные данные заданы некорректно.
