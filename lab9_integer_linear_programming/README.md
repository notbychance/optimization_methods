# Лабораторная работа №9. Целочисленное линейное программирование

Программа решает задачи целочисленного линейного программирования методом ветвей и границ. Алгоритм реализован самостоятельно, без библиотек оптимизации.

## Состав проекта

```text
lab9_integer_linear_programming_python/
├── main.py
├── branch_and_bound.py
├── simplex.py
├── linear_problem.py
├── fraction_tools.py
├── problem_reader.py
├── snapshot.py
├── README.md
└── examples/
    ├── lab9_integer_example.json
    └── lab9_binary_example.json
```

Назначение модулей:

```text
branch_and_bound.py — собственный метод ветвей и границ;
simplex.py          — собственный двухфазный симплекс-метод для LP-релаксаций;
snapshot.py         — поэтапный вывод дерева ветвления и симплекс-таблиц в Markdown-файл;
problem_reader.py   — чтение условия задачи из JSON;
linear_problem.py   — модели данных задачи, ограничений, узлов и результатов;
fraction_tools.py   — работа с рациональными числами;
main.py             — модуль запуска программы.
```

## Требования

Нужен только Python 3.10 или новее. Внешние библиотеки не используются.

Проверка версии Python:

```bash
python --version
```

## Запуск

Из каталога проекта:

```bash
python main.py --input examples/lab9_integer_example.json --snapshot integer_lp_snapshot.md
```

Сокращенный вариант:

```bash
python main.py -i examples/lab9_integer_example.json -s integer_lp_snapshot.md
```

После запуска программа выводит результат в консоль и создает файл снимков алгоритма.

## Формат входного файла

Входной файл задается в формате JSON.

```json
{
  "objective": {
    "type": "max",
    "coefficients": [3, 2]
  },
  "variable_names": ["x1", "x2"],
  "variable_types": ["integer", "integer"],
  "constraints": [
    {
      "name": "resource_1",
      "coefficients": [2, 1],
      "sign": "<=",
      "rhs": 7
    },
    {
      "name": "resource_2",
      "coefficients": [1, 2],
      "sign": "<=",
      "rhs": 7
    }
  ],
  "max_nodes": 100
}
```

### Поля JSON

`objective.type` — тип целевой функции:

```text
max — максимизация;
min — минимизация.
```

`objective.coefficients` — коэффициенты целевой функции.

`variable_names` — имена переменных. Если поле не задано, будут использованы имена `x1`, `x2`, ...

`variable_types` — типы переменных:

```text
integer    — целочисленная переменная;
binary     — двоичная переменная, автоматически добавляется ограничение x <= 1;
continuous — непрерывная переменная, не участвует в ветвлении.
```

Если `variable_types` не задано, все переменные считаются целочисленными.

`constraints` — список ограничений. Для каждого ограничения указываются:

```text
name         — необязательное имя ограничения;
coefficients — коэффициенты левой части;
sign         — знак ограничения: <=, >= или =;
rhs          — правая часть ограничения.
```

`max_nodes` — максимальное число узлов дерева ветвей и границ.

## Пример результата

Для файла `examples/lab9_integer_example.json` задача имеет LP-релаксацию с дробным решением. Метод ветвей и границ строит дополнительные ограничения и находит целочисленный оптимум.

Пример вывода:

```text
Статус: optimal
Сообщение: Оптимальное целочисленное решение найдено методом ветвей и границ.
Количество рассмотренных узлов: 5
Значение целевой функции: 11
Значения переменных:
  x1 = 3
  x2 = 1
Файл со снимками: integer_lp_snapshot.md
```

## Файл снимков алгоритма

Файл `integer_lp_snapshot.md` содержит:

1. исходную математическую модель;
2. типы переменных;
3. ограничения исходной задачи;
4. каждый узел дерева ветвей и границ;
5. дополнительные ограничения узла;
6. симплекс-таблицы LP-релаксации по фазам I и II;
7. результат LP-релаксации узла;
8. причину отсечения узла;
9. выбранную переменную ветвления;
10. обновление текущего рекорда;
11. итоговое целочисленное решение.

## Суть реализованного метода

Исходная задача целочисленного линейного программирования заменяется LP-релаксацией: требование целочисленности временно отбрасывается. LP-релаксация решается собственным двухфазным симплекс-методом.

Если решение LP-релаксации целочисленное, оно становится кандидатом на оптимум. Если хотя бы одна целочисленная переменная получила дробное значение, строятся две ветви:

```text
xj <= floor(xj*)
xj >= ceil(xj*)
```

Каждая ветвь образует новую LP-релаксацию. Узлы отбрасываются, если они несовместны или их оценка не может улучшить текущий рекорд.

## Поддерживаемые статусы

```text
optimal    — найдено оптимальное целочисленное решение;
infeasible — целочисленное допустимое решение не найдено;
unbounded  — LP-релаксация оказалась неограниченной;
node_limit — достигнут предел числа узлов дерева.
```
