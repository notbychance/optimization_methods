# Модуль 9: целочисленное линейное программирование

Консольная программа для решения задач целочисленного линейного программирования методом ветвей и границ. LP-релаксации решаются встроенным двухфазным симплекс-методом. Оптимизационные библиотеки не используются.

## Возможности

- задачи на максимум и минимум;
- ограничения `<=`, `>=`, `=`;
- типы переменных `integer`, `binary`, `continuous`;
- метод ветвей и границ;
- симплекс-метод для LP-релаксаций;
- отсечение узлов по недопустимости, неограниченности, границе и целочисленности;
- ограничение максимального количества узлов через `max_nodes`;
- точные вычисления через `fractions.Fraction`;
- Markdown-протокол дерева ветвления и симплексных таблиц.

## Запуск

Из каталога модуля:

```bash
python main.py -i examples/lab9_example.json -s integer_lp_snapshot.md
```

Из корня проекта:

```bash
python main.py -n 9 -i lab9_example.json -o results/lab9.md
```

## Формат JSON

```json
{
  "objective": {
    "type": "max",
    "coefficients": [3, 2]
  },
  "variable_names": ["x1", "x2"],
  "variable_types": ["integer", "integer"],
  "constraints": [
    {
      "name": "resource_1",
      "coefficients": [2, 1],
      "sign": "<=",
      "rhs": 7
    },
    {
      "name": "resource_2",
      "coefficients": [1, 2],
      "sign": "<=",
      "rhs": 5
    }
  ],
  "max_nodes": 100
}
```

Поля:

| Поле | Описание |
|---|---|
| `objective.type` | `max` или `min` |
| `objective.coefficients` | коэффициенты целевой функции |
| `variable_names` | имена переменных; необязательно |
| `variable_types` | типы переменных; если поле отсутствует, все переменные считаются `integer` |
| `constraints` | список ограничений |
| `constraints[].coefficients` | коэффициенты ограничения |
| `constraints[].sign` | `<=`, `>=` или `=` |
| `constraints[].rhs` | правая часть ограничения |
| `max_nodes` | лимит количества узлов дерева; необязательно |

Типы переменных:

| Тип | Описание |
|---|---|
| `integer` | целочисленная переменная |
| `binary` | двоичная переменная, дополнительно ограниченная диапазоном 0..1 |
| `continuous` | непрерывная переменная |

## Результат

В консоль выводятся статус, сообщение, количество рассмотренных узлов, значение целевой функции и значения переменных. Markdown-файл содержит исходную модель, LP-релаксации, симплекс-таблицы, ветвления, отсечения и итоговое решение.

## Возможные статусы

| Статус | Смысл |
|---|---|
| `optimal` | найдено оптимальное целочисленное решение |
| `infeasible` | допустимое целочисленное решение не найдено |
| `unbounded` | релаксация не ограничена |
| `node_limit` | достигнут лимит узлов дерева |
