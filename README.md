# Лабораторные работы по методам оптимизации

Репозиторий содержит набор консольных Python-программ для выполнения лабораторных работ по дисциплине **«Методы оптимизации»**. Каждая лабораторная работа реализована отдельным модулем, а в корне проекта расположен общий файл запуска `main.py`, который позволяет выбрать номер работы, входной JSON-файл и файл экспорта результата.

Все алгоритмы реализованы вручную, без готовых библиотек оптимизации. Для вычислений используются стандартные возможности Python, в том числе рациональные числа через `fractions.Fraction`, где это требуется для уменьшения ошибок округления.

## Требования

Для запуска нужен **Python 3.10 или новее**.

Проверка версии Python:

```bash
python --version
```

Внешние зависимости устанавливать не нужно: проект использует только стандартную библиотеку Python.

## Общий запуск из корня репозитория

Основной способ запуска — через общий файл `main.py` в корне проекта.

Общий формат команды:

```bash
python main.py --number НОМЕР_РАБОТЫ --input ФАЙЛ_ИМПОРТА.json --output ФАЙЛ_ЭКСПОРТА.md
```

Короткая форма:

```bash
python main.py -n НОМЕР_РАБОТЫ -i ФАЙЛ_ИМПОРТА.json -o ФАЙЛ_ЭКСПОРТА.md
```

Параметры:

| Параметр | Короткая форма | Описание |
|---|---|---|
| `--number` | `-n` | Номер лабораторной работы от 2 до 9. |
| `--input` | `-i` | Путь к JSON-файлу с условием задачи. |
| `--import-file` | — | Альтернативное имя параметра для файла импорта. |
| `--snapshot` | `-s` | Путь к Markdown-файлу для экспорта снимков решения. |
| `--output` | `-o` | Альтернативное имя параметра для файла экспорта. |
| `--export` | — | Альтернативное имя параметра для файла экспорта. |
| `--export-file` | — | Альтернативное имя параметра для файла экспорта. |

Если параметры не переданы, общий `main.py` запросит их интерактивно:

```bash
python main.py
```

После запуска программа:

1. принимает номер лабораторной работы;
2. определяет соответствующий каталог модуля;
3. находит входной JSON-файл;
4. запускает `main.py` выбранной лабораторной работы;
5. сохраняет подробный протокол решения в Markdown-файл.

## Поиск входного файла

Если для `--input` указан относительный путь, общий запуск ищет файл в нескольких местах:

1. в текущем рабочем каталоге;
2. в корне репозитория;
3. в каталоге выбранной лабораторной работы;
4. в каталоге `examples` выбранной лабораторной работы.

Поэтому можно запускать примеры коротко, указывая только имя файла из `examples`.

Например:

```bash
python main.py -n 5 -i lab5_shortest_path_example.json -o results/lab5_shortest_path.md
```

## Запуск отдельной лабораторной работы напрямую

Каждую лабораторную работу можно запускать отдельно из её каталога.

Пример для ЛР 2:

```bash
cd lab2_simplex
python main.py --input examples/lab2_example.json --snapshot simplex_snapshot.md
```

Пример для ЛР 9:

```bash
cd lab9_integer_linear_programming
python main.py --input examples/lab9_integer_example.json --snapshot integer_lp_snapshot.md
```

Общий запуск из корня удобнее, если нужно работать с несколькими лабораторными работами через единый интерфейс.

## Формат входных данных

Входные данные задаются в формате **JSON**. Формат зависит от выбранной лабораторной работы.

Кратко:

- ЛР 2, 3, 7, 9 используют модели линейного программирования с целевой функцией, переменными и ограничениями;
- ЛР 4 использует данные транспортной задачи: запасы, потребности и матрицу стоимостей;
- ЛР 5 использует описание графа и тип сетевой задачи;
- ЛР 6 использует данные для задачи о рюкзаке или универсальной многоэтапной задачи;
- ЛР 8 использует платежные матрицы, матрицы решений или биматричные игры.

Подробное описание формата входных данных приведено в `README.md` соответствующей лабораторной работы.

## Формат результата

Результат работы состоит из двух частей:

1. краткий итог выводится в консоль;
2. подробный протокол решения сохраняется в Markdown-файл, указанный через `--snapshot`, `--output`, `--export` или `--export-file`.

Markdown-файл можно использовать как основу для отчета по лабораторной работе: он содержит промежуточные таблицы, вычислительные шаги, итоговый статус и найденное решение.

## Рекомендуемый порядок работы

1. Выбрать номер лабораторной работы.
2. Открыть `README.md` нужного каталога и изучить формат JSON.
3. Скопировать пример из каталога `examples` или подготовить собственный входной файл.
4. Запустить общий `main.py` из корня репозитория.
5. Проверить консольный вывод и Markdown-файл с протоколом решения.
6. Перенести необходимые таблицы и выводы в отчет.

## Возможные ошибки запуска

### Не найден входной JSON-файл

Проверьте путь, переданный в `--input`. Можно указать абсолютный путь или имя файла из каталога `examples` выбранной лабораторной работы.

Пример корректного запуска:

```bash
python main.py -n 4 -i lab4_transport_example.json -o results/lab4.md
```

### Неверный номер лабораторной работы

Допустимые номера: `2`, `3`, `4`, `5`, `6`, `7`, `8`, `9`.

### Файл экспорта не создается

Проверьте, что каталог для файла экспорта существует или может быть создан. Общий `main.py` автоматически создает родительский каталог для файла экспорта, если это возможно.

Например, команда создаст каталог `results`, если его ещё нет:

```bash
python main.py -n 2 -i lab2_example.json -o results/lab2.md
```

## Примечания по реализации общего запуска

Корневой `main.py` не импортирует модули лабораторных работ напрямую. Он запускает `main.py` выбранной лабораторной работы отдельным процессом. Это сделано для того, чтобы избежать конфликтов между одноимёнными файлами разных лабораторных работ, например `problem_reader.py`, `snapshot.py`, `linear_problem.py`.

Такой подход сохраняет независимость модулей и позволяет запускать все лабораторные работы через единый интерфейс.
