# Модуль 2: двухфазный симплекс-метод

Консольная программа для решения задач линейного программирования собственным двухфазным симплекс-методом. Программа читает модель из JSON-файла, строит симплекс-таблицы, выполняет фазу поиска допустимого базиса при необходимости и сохраняет пошаговый протокол в Markdown.

## Возможности

- задачи на максимум и минимум;
- ограничения `<=`, `>=`, `=`;
- автоматическое добавление остаточных, избыточных и искусственных переменных;
- фаза I для поиска допустимого опорного решения;
- фаза II для оптимизации исходной целевой функции;
- точные вычисления через `fractions.Fraction`;
- Markdown-снимки всех ключевых таблиц;
- статусы `optimal`, `infeasible`, `unbounded`.

├── simplex.py
├── snapshot.py

| Файл | Назначение |
|---|---|
| `main.py` | Разбор аргументов, запуск чтения, решения и вывода результата |
| `problem_reader.py` | Чтение JSON и преобразование чисел в дроби |
| `simplex.py` | Реализация двухфазного симплекс-метода |
| `snapshot.py` | Запись симплекс-таблиц в Markdown |
| `examples/` | Примеры входных данных |

## Запуск

Из каталога модуля:

```bash
python main.py --input examples/lab2_example.json --snapshot simplex_snapshot.md
```

Короткая форма:

```bash
python main.py -i examples/lab2_example.json -s simplex_snapshot.md
```

Из корня проекта:

```bash
python main.py -n 2 -i lab2_example.json -o results/lab2_simplex.md
```

## Формат JSON

```json
{
  "objective": {
    "type": "min",
    "coefficients": [830, 835, 360, 450]
  },
  "variable_names": ["D1", "D2", "D3", "D4"],
  "constraints": [
    {
      "name": "constraint_1",
      "coefficients": [10, 18, 20, 15],
      "sign": ">=",
      "rhs": 800
    }
  ]
}
```

Поля:

| Поле | Описание |
|---|---|
| `objective.type` | `max` или `min` |
| `objective.coefficients` | коэффициенты целевой функции |
| `variable_names` | имена исходных переменных; поле необязательное |
| `constraints` | список ограничений |
| `constraints[].coefficients` | коэффициенты ограничения |
| `constraints[].sign` | один из знаков: `<=`, `>=`, `=` |
| `constraints[].rhs` | правая часть ограничения |
| `constraints[].name` | имя ограничения; поле необязательное |

Числа можно задавать как целые, десятичные или строки-дроби, например `"3/5"`.

## Результат

В консоль выводятся:

- статус;
- сообщение;
- количество итераций;
- значение целевой функции для оптимального решения;
- значения исходных переменных.

Markdown-файл содержит:

- исходную таблицу фазы I или фазы II;
- переходы между фазами;
- таблицы после каждой итерации;
- итоговую таблицу и найденное решение.
