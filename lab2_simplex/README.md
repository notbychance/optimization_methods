# Лабораторная работа № 2: симплекс-метод на Python

Программа предназначена для решения задач линейного программирования симплекс-методом. В проекте используется собственная реализация алгоритма без библиотечных оптимизаторов. Программа читает условие задачи из JSON-файла, решает задачу двухфазным симплекс-методом и сохраняет поэтапные снимки симплекс-таблиц в отдельный Markdown-файл.

Проект подготовлен для второй лабораторной работы по дисциплине «Методы оптимизации», где требуется составить математическую модель задачи, решить её симплекс-методом, разработать программу общего вида и составить отчет.

## Возможности программы

- решение задач линейного программирования на максимум и минимум;
- поддержка ограничений вида `<=`, `>=`, `=`;
- автоматическое добавление остаточных, избыточных и искусственных переменных;
- двухфазный симплекс-метод для поиска начального допустимого базисного решения;
- работа с обыкновенными дробями через `fractions.Fraction`, что уменьшает ошибки округления;
- чтение условия задачи из JSON-файла;
- сохранение всех симплекс-таблиц по шагам в Markdown-файл;
- вывод итогового статуса, значения целевой функции и значений исходных переменных в консоль.

## Быстрый запуск

Перейдите в каталог проекта:

```bash
cd lab2_simplex_python
```

Запустите программу на примере из каталога `examples`:

```bash
python main.py --input examples/lab2_example.json --snapshot simplex_snapshot.md
```

Короткая форма запуска:

```bash
python main.py -i examples/lab2_example.json -s simplex_snapshot.md
```

После запуска программа выведет результат в консоль и создаст файл `simplex_snapshot.md` со всеми симплекс-таблицами.

## Формат входного файла

Входной файл должен быть в формате JSON.

Общая структура:

```json
{
  "objective": {
    "type": "max",
    "coefficients": [1, 2, 1, -1, 0]
  },
  "variable_names": ["x1", "x2", "x3", "x4", "x5"],
  "constraints": [
    {
      "coefficients": [-1, 5, 1, 1, 1],
      "sign": "=",
      "rhs": 10
    },
    {
      "coefficients": [-2, 1, -1, 3, 0],
      "sign": "=",
      "rhs": -6
    },
    {
      "coefficients": [0, 10, 1, 2, 3],
      "sign": "=",
      "rhs": 25
    }
  ]
}
```

Описание полей:

| Поле | Описание |
|---|---|
| `objective` | Описание целевой функции. |
| `objective.type` | Тип оптимизации: `max` для максимизации или `min` для минимизации. |
| `objective.coefficients` | Коэффициенты целевой функции. |
| `variable_names` | Имена исходных переменных. Поле необязательное. Если его не указать, будут использованы имена `x1`, `x2`, `x3` и далее. |
| `constraints` | Список ограничений задачи. |
| `constraints[].coefficients` | Коэффициенты левой части ограничения. |
| `constraints[].sign` | Знак ограничения: `<=`, `>=` или `=`. |
| `constraints[].rhs` | Правая часть ограничения. |

Количество коэффициентов в каждом ограничении должно совпадать с количеством коэффициентов целевой функции.

## Поддерживаемые обозначения

Для типа целевой функции используются значения:

```text
max
min
```

Для знаков ограничений используются значения:

```text
<=
>=
=
```

Также при чтении файла поддерживаются альтернативные записи:

```text
<
≤
>
≥
==
```

Числа можно задавать как целые, вещественные или строковые дроби:

```json
[1, 2.5, "3/4", "2,5"]
```

Строка `"2,5"` будет прочитана как `2.5`.

## Пример входного файла

Файл `examples/lab2_example.json` содержит пример задачи:

```json
{
  "objective": {
    "type": "max",
    "coefficients": [1, 2, 1, -1, 0]
  },
  "variable_names": ["x1", "x2", "x3", "x4", "x5"],
  "constraints": [
    {
      "coefficients": [-1, 5, 1, 1, 1],
      "sign": "=",
      "rhs": 10
    },
    {
      "coefficients": [-2, 1, -1, 3, 0],
      "sign": "=",
      "rhs": -6
    },
    {
      "coefficients": [0, 10, 1, 2, 3],
      "sign": "=",
      "rhs": 25
    }
  ]
}
```

Эта задача соответствует форме:

```text
L = x1 + 2x2 + x3 - x4 -> max

-x1 + 5x2 + x3 + x4 + x5 = 10
-2x1 + x2 - x3 + 3x4 = -6
10x2 + x3 + 2x4 + 3x5 = 25

x1, x2, x3, x4, x5 >= 0
```

## Файл снимков симплекс-таблиц

Файл снимков создается модулем `snapshot.py`. По умолчанию он называется `simplex_snapshot.md`.

В этот файл записываются:

- исходная таблица фазы I;
- каждая итерация фазы I;
- сообщение о завершении фазы I;
- таблица перехода к фазе II;
- каждая итерация фазы II;
- итоговая симплекс-таблица.

Формат снимка:

```markdown
## Фаза I. Итерация 1

- Фаза: I
- Номер итерации внутри фазы: 1
- Пояснение: В базис введена переменная x2; из базиса выведена переменная r1.

| Базис | x1 | x2 | x3 | x4 | x5 | r1 | r2 | r3 | Свободные члены |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| x2 | -1/5 | 1 | 1/5 | 1/5 | 1/5 | 1/5 | 0 | 0 | 2 |
```

Точный набор столбцов зависит от исходной задачи, количества ограничений и необходимости вводить дополнительные переменные.

## Логика работы алгоритма

Программа использует двухфазный симплекс-метод.

### Фаза I

Фаза I нужна для поиска начального допустимого базисного решения.

На этом этапе программа:

1. нормализует ограничения с отрицательной правой частью;
2. добавляет остаточные переменные для ограничений `<=`;
3. добавляет избыточные и искусственные переменные для ограничений `>=`;
4. добавляет искусственные переменные для ограничений `=`;
5. строит вспомогательную целевую функцию `W`;
6. максимизирует `W`;
7. проверяет, равно ли оптимальное значение `W` нулю.

Если значение `W` не равно нулю, исходная система ограничений считается несовместной.

### Фаза II

Фаза II решает исходную задачу.

На этом этапе программа:

1. удаляет искусственные переменные;
2. восстанавливает исходную целевую функцию;
3. продолжает симплекс-итерации;
4. останавливается, когда в строке целевой функции нет отрицательных коэффициентов;
5. выводит оптимальные значения исходных переменных и значение целевой функции.

## Правило выбора ведущего элемента

В программе используется следующее правило:

1. Ведущий столбец выбирается по наиболее отрицательному коэффициенту в строке целевой функции.
2. Ведущая строка выбирается по минимальному положительному отношению свободного члена к элементу ведущего столбца.
3. После выбора ведущего элемента выполняется пересчет таблицы методом Жордана-Гаусса.

## Статусы результата

Программа может вернуть один из следующих статусов:

| Статус | Значение |
|---|---|
| `optimal` | Оптимальное решение найдено. |
| `infeasible` | Система ограничений несовместна, допустимого решения нет. |
| `unbounded` | Целевая функция не ограничена на множестве допустимых решений. |

## Ограничения программы

- Все исходные переменные считаются неотрицательными.
- Программа решает задачи линейного программирования в вещественной постановке, без требования целочисленности переменных.
- Программа не использует готовые оптимизационные библиотеки.
- Вырожденные задачи могут потребовать дополнительных антициклических правил, например правила Бланда. В текущей версии используется стандартный выбор по наиболее отрицательному коэффициенту.

## Как добавить собственную задачу

Создайте новый JSON-файл, например `examples/my_problem.json`:

```json
{
  "objective": {
    "type": "max",
    "coefficients": [5, 4]
  },
  "variable_names": ["x1", "x2"],
  "constraints": [
    {
      "coefficients": [6, 4],
      "sign": "<=",
      "rhs": 24
    },
    {
      "coefficients": [1, 2],
      "sign": "<=",
      "rhs": 6
    },
    {
      "coefficients": [-1, 1],
      "sign": "<=",
      "rhs": 1
    },
    {
      "coefficients": [0, 1],
      "sign": "<=",
      "rhs": 2
    }
  ]
}
```

Запустите программу:

```bash
python main.py --input examples/my_problem.json --snapshot my_problem_snapshot.md
```
