# Лабораторная работа №3. Двойственный симплекс-метод и матричное представление симплексных решений

## Назначение программы

Программа решает задачу линейного программирования двойственным симплекс-методом без использования библиотечных оптимизаторов. Дополнительно программа сохраняет поэтапные снимки алгоритма в Markdown-файл: исходную таблицу, каждую итерацию, итоговую таблицу и матричное представление текущего базисного решения.

Программа подходит для оформления лабораторной работы №3 по теме «Двойственный симплекс-метод. Матричное представление симплексных решений».

## Возможности

1. Решение задачи линейного программирования двойственным симплекс-методом.
2. Работа с точными рациональными числами через `fractions.Fraction`.
3. Чтение условия из JSON-файла.
4. Поддержка двух режимов входных данных:
   - `model` — ввод математической модели: целевая функция и ограничения;
   - `tableau` — ввод готовой симплекс-таблицы.
5. Вывод результата в консоль.
6. Запись снимков итераций в файл.
7. Вывод матричного представления на каждой итерации:
   - матрица базиса `B`;
   - обратная матрица `B^(-1)`;
   - базисное решение `X_B = B^(-1) * b`;
   - преобразованная матрица `B^(-1) * A`;
   - оценки `z_j - c_j`.

## Описание модулей

### `main.py`

Модуль запуска. Обрабатывает аргументы командной строки, вызывает чтение входного файла, запускает решатель и выводит результат.

### `problem_reader.py`

Модуль чтения условия из JSON-файла. Поддерживает два режима: `model` и `tableau`.

### `dual_simplex.py`

Основной модуль двойственного симплекс-метода. Выполняет построение канонической таблицы, проверку двойственной допустимости, выбор исключаемой и вводимой переменной, пересчет таблицы методом Жордана-Гаусса и формирование итогового решения.

### `matrix_tools.py`

Модуль матричных операций: умножение матриц, умножение матрицы на вектор, выбор столбцов, транспонирование, построение обратной матрицы методом Жордана-Гаусса.

### `snapshot.py`

Модуль поэтапного вывода таблиц и матричных представлений в Markdown-файл.

### `linear_problem.py`

Модуль с классами данных для математической модели, канонической формы и результата решения.

## Быстрый запуск

Из папки проекта выполните:

```bash
python main.py --input examples/lab3_dual_example.json --snapshot dual_simplex_snapshot.md
```

После запуска в консоль будет выведен результат, а файл `dual_simplex_snapshot.md` будет содержать все таблицы и матричные представления.

## Пример входного файла в режиме `model`

Файл `examples/lab3_dual_example.json`:

```json
{
  "mode": "model",
  "objective": {
    "type": "min",
    "coefficients": [2, 3]
  },
  "variable_names": ["x1", "x2"],
  "constraints": [
    {
      "coefficients": [1, 1],
      "sign": ">=",
      "rhs": 4
    },
    {
      "coefficients": [2, 1],
      "sign": ">=",
      "rhs": 6
    }
  ]
}
```

Математическая модель примера:

```text
Минимизировать F = 2x1 + 3x2
при ограничениях:
    x1 + x2 >= 4
   2x1 + x2 >= 6
    x1, x2 >= 0
```

Для двойственного симплекс-метода ограничения вида `>=` преобразуются так, чтобы начальная таблица имела отрицательные свободные члены, но неотрицательные оценки в строке целевой функции.

## Пример входного файла в режиме `tableau`

Режим `tableau` нужен, если начальная симплекс-таблица уже составлена вручную.

Файл `examples/lab3_tableau_example.json`:

```json
{
  "mode": "tableau",
  "objective_type": "min",
  "column_names": ["x1", "x2", "s1", "s2"],
  "basis": ["s1", "s2"],
  "rows": [
    [-1, -1, 1, 0, -4],
    [-2, -1, 0, 1, -6]
  ],
  "objective_row": [2, 3, 0, 0, 0],
  "max_costs": [-2, -3, 0, 0],
  "original_objective": [2, 3],
  "original_variable_count": 2
}
```

Поля режима `tableau`:

| Поле | Назначение |
|---|---|
| `column_names` | имена всех переменных таблицы |
| `basis` | список текущих базисных переменных |
| `rows` | строки ограничений, последний элемент каждой строки — свободный член |
| `objective_row` | строка оценок `z_j - c_j`, последний элемент — значение целевой функции в max-форме |
| `max_costs` | коэффициенты целевой функции в max-форме |
| `original_objective` | коэффициенты исходной целевой функции |
| `original_variable_count` | количество исходных переменных, без дополнительных |

## Содержимое файла снимков

Файл снимков содержит разделы вида:

```text
## Исходная таблица двойственного симплекс-метода

### Симплекс-таблица
...

### Матричное представление текущего базисного решения
...
```

Для каждой итерации записываются:

1. текущая симплекс-таблица;
2. пояснение, какая переменная вводится в базис и какая выводится;
3. матрица базиса `B`;
4. обратная матрица `B^(-1)`;
5. вектор базисного решения `X_B`;
6. преобразованная матрица `B^(-1) * A`;
7. строка оценок `z_j - c_j`.

## Логика двойственного симплекс-метода

В программе используется следующая схема:

1. Проверяется двойственная допустимость начальной таблицы: все оценки `z_j - c_j` должны быть неотрицательными.
2. Если все свободные члены также неотрицательны, текущее решение является оптимальным.
3. Если есть отрицательные свободные члены, выбирается строка с наиболее отрицательным свободным членом.
4. В выбранной строке выбирается вводимый столбец среди отрицательных коэффициентов.
5. Для вводимого столбца используется отношение:

```text
(z_j - c_j) / (-a_ij)
```

6. Выполняется пересчет таблицы методом Жордана-Гаусса.
7. Процесс повторяется до получения неотрицательных свободных членов или до вывода о несовместности задачи.

## Статусы результата

| Статус | Значение |
|---|---|
| `optimal` | оптимальное решение найдено |
| `not_dual_feasible` | начальная таблица не подходит для двойственного симплекс-метода, так как строка оценок содержит отрицательные элементы |
| `infeasible` | допустимое решение отсутствует |

## Как адаптировать под свой вариант

1. Откройте файл `examples/lab3_dual_example.json`.
2. Замените коэффициенты целевой функции.
3. Замените ограничения на данные своего варианта.
4. Укажите `type`: `min` или `max`.
5. Запустите программу.
6. Используйте консольный результат и файл снимков в отчете.

Для задач, где начальная модель не дает двойственно допустимую таблицу, составьте начальную таблицу вручную и используйте режим `tableau`.
