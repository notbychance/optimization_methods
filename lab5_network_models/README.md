# Лабораторная работа №5 — сетевые модели

Программа решает типовые задачи сетевых моделей без использования специализированных библиотек оптимизации. Дополнительно в модуль добавлена поддержка **17-го варианта** из общего комплекса: проверка предвыигрышной конфигурации на поле крестиков-ноликов.

## Реализованные режимы

| `problem_type` | Назначение | Алгоритм |
|---|---|---|
| `mst` | минимальное остовное дерево | алгоритм Краскала |
| `shortest_path` | кратчайший путь | алгоритм Дейкстры |
| `max_flow` | максимальный поток | алгоритм Эдмондса-Карпа |
| `min_cost_flow` | поток наименьшей стоимости | последовательный выбор кратчайшего по стоимости увеличивающего пути |
| `prewinning_tictactoe_configuration` | 17-й вариант: проверка предвыигрышной конфигурации крестиков-ноликов | полный перебор допустимых ходов и проверка цепочек по 4 направлениям |

Программа соответствует структуре лабораторной работы: выполняет математическую обработку модели, решает задачу алгоритмом общего вида, читает исходные данные из файла и сохраняет пошаговые снимки вычислений.

## Запуск

Общий вид команды при запуске из каталога `lab5_network_models`:

```bash
python main.py --input путь_к_json --snapshot путь_к_файлу_снимков.md
```

Примеры для типовых сетевых режимов:

```bash
python main.py --input examples/lab5_mst_example.json --snapshot mst_snapshot.md
python main.py --input examples/lab5_shortest_path_example.json --snapshot shortest_path_snapshot.md
python main.py --input examples/lab5_max_flow_example.json --snapshot max_flow_snapshot.md
python main.py --input examples/lab5_min_cost_flow_example.json --snapshot min_cost_flow_snapshot.md
```

Пример для конкретной матрицы 17-го варианта:

```bash
python main.py --input examples/lab5_tictactoe_concrete_example.json --snapshot tictactoe_snapshot.md
```

При запуске из корня общего комплекса используется корневой `main.py`:

```bash
python main.py -n 5 -i lab5_tictactoe_concrete_example.json -o results/lab5_tictactoe_concrete_example.md
```

## Формат входного файла для сетевых задач

Входной файл имеет формат JSON.

Общие поля для режимов `mst`, `shortest_path`, `max_flow`, `min_cost_flow`:

| Поле | Обязательность | Описание |
|---|---:|---|
| `problem_type` | да | тип задачи: `mst`, `shortest_path`, `max_flow`, `min_cost_flow` |
| `nodes` | да | список вершин сети |
| `edges` | да | список ребер или дуг |
| `directed` | нет | признак ориентированного графа |
| `source` | для 3 режимов | исходная вершина для `shortest_path`, `max_flow`, `min_cost_flow` |
| `target` | для 3 режимов | конечная вершина для `shortest_path`, `max_flow`, `min_cost_flow` |
| `desired_flow` | нет | требуемая величина потока для `min_cost_flow` |

Поля ребра/дуги:

| Поле | Описание |
|---|---|
| `from` | начальная вершина |
| `to` | конечная вершина |
| `weight` | вес ребра или длина дуги |
| `capacity` | пропускная способность дуги |
| `cost` | стоимость передачи единицы потока по дуге |

## Формат входного файла для 17-го варианта

Для 17-го варианта используется режим:

```text
prewinning_tictactoe_configuration
```

Задача: дано клеточное поле `m × n`, полученное после игры в крестики-нолики на бесконечном поле. Нужно проверить, является ли конфигурация предвыигрышной для одного из игроков. Конфигурация предвыигрышная, если существует ход в пустую клетку, после которого по горизонтали, вертикали, главной диагонали или побочной диагонали появляется цепочка из пяти одинаковых знаков.

Поля JSON:

| Поле | Обязательность | Описание |
|---|---:|---|
| `problem_type` | да | должно быть `prewinning_tictactoe_configuration` |
| `win_length` | нет | длина победной цепочки, по умолчанию `5` |
| `symbols.first_player` | нет | символ первого игрока, по умолчанию `X` |
| `symbols.second_player` | нет | символ второго игрока, по умолчанию `O` |
| `symbols.empty_cell` | нет | символ пустой клетки, по умолчанию `.` |
| `board` | да для расчёта | поле как список строк одинаковой длины |

Пример конкретного поля:

```json
{
  "problem_type": "prewinning_tictactoe_configuration",
  "win_length": 5,
  "symbols": {
    "first_player": "X",
    "second_player": "O",
    "empty_cell": "."
  },
  "board": [
    ".......",
    "..XXXX.",
    "...O...",
    "...O...",
    "...O...",
    "......."
  ]
}
```

В этом примере программа найдет предвыигрышные ходы для `X`: можно поставить знак слева или справа от цепочки `XXXX` и получить пять знаков подряд.

## Особенность примеров 17-го варианта

В методических указаниях для 17-го варианта ЛР 5 задано общее условие, но конкретная заполненная матрица поля не приведена. Поэтому файлы:

```text
lab5_mst_example.json
lab5_shortest_path_example.json
lab5_max_flow_example.json
lab5_min_cost_flow_example.json
```

в общем комплексе сохранены как заготовки с условием 17-го варианта. В них поле `board` оставлено текстовым пояснением. Такие файлы запускаются без аварийной ошибки, но возвращают статус:

```text
input_required
```

Это означает, что для полного расчёта нужно заменить поле `board` на список строк.

Для проверки работоспособности алгоритма добавлен отдельный файл:

```text
lab5_tictactoe_concrete_example.json
```

В нем уже есть конкретная матрица поля.

## Файл снимков алгоритма

Файл снимков создается в формате Markdown.

Для сетевых задач в него записываются:

- исходные матрицы весов, пропускных способностей или стоимостей;
- таблицы выбора ребер для алгоритма Краскала;
- таблицы меток и релаксаций для алгоритма Дейкстры;
- матрицы текущего потока и остаточной сети для задач потока;
- итоговые таблицы с найденным решением.

Для 17-го варианта в файл записываются:

1. исходная матрица поля;
2. таблица всех проверенных пустых клеток для обоих игроков;
3. найденные предвыигрышные ходы;
4. матрица поля после каждого найденного предвыигрышного хода;
5. пояснение `input_required`, если конкретное поле `board` не задано.

## Статусы результата

| Статус | Значение |
|---|---|
| `optimal` | сетевое решение найдено |
| `disconnected` | минимальное остовное дерево не построено, граф несвязный |
| `unreachable` | путь между заданными вершинами не найден |
| `infeasible` | требуемый поток невозможно передать через сеть |
| `input_required` | для 17-го варианта не задана конкретная матрица `board` |
| `prewinning` | конфигурация является предвыигрышной |
| `not_prewinning` | предвыигрышных ходов не найдено |
