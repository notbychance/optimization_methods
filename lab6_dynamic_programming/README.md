# Лабораторная работа 6. Детерминированные модели динамического программирования

Программа предназначена для выполнения лабораторной работы по теме **«Детерминированные модели динамического программирования»**.

В методических указаниях для этой лабораторной работы требуется решить математическую модель своего варианта, разработать программу общего вида и составить отчет по работе. Динамическое программирование рассматривается как разбиение многомерной задачи на этапы, где на каждом этапе решается одномерная подзадача, а ключевыми элементами модели являются этапы, состояния и альтернативы.

## Возможности программы

Программа поддерживает два вида детерминированных моделей:

1. `knapsack` — задача о загрузке, то есть задача выбора количества предметов при ограниченной грузоподъемности.
2. `finite_horizon` — универсальная конечная многоэтапная задача с состояниями, альтернативами, переходами и граничными значениями.

Алгоритмы реализованы вручную, без библиотек оптимизации.

## Требования

Нужен только Python 3.10 или выше.

Сторонние библиотеки не требуются.

## Запуск

Для задачи о загрузке:

```bash
python main.py --input examples/lab6_knapsack_example.json --snapshot knapsack_snapshot.md
```

Для универсальной многоэтапной задачи:

```bash
python main.py --input examples/lab6_finite_horizon_example.json --snapshot finite_horizon_snapshot.md
```

Параметры запуска:

```text
--input, -i     путь к JSON-файлу с условием задачи
--snapshot, -s  путь к файлу, куда будут записаны таблицы этапов
```

## Формат входного файла для задачи о загрузке

```json
{
  "problem_type": "knapsack",
  "optimization": "max",
  "capacity": 10,
  "items": [
    {
      "name": "A",
      "weight": 2,
      "profit": 6,
      "max_count": 3
    },
    {
      "name": "B",
      "weight": 3,
      "profit": 8,
      "max_count": 2
    },
    {
      "name": "C",
      "weight": 5,
      "profit": 14
    }
  ]
}
```

Поля:

```text
problem_type  — тип задачи, для задачи о загрузке: knapsack
optimization  — критерий оптимизации: max или min
capacity      — грузоподъемность, целое число
items         — список предметов
name          — имя предмета
weight        — вес одной единицы предмета
profit        — прибыль или значение одной единицы предмета
max_count     — максимальное количество предметов данного вида; если поле не задано, количество не ограничено вместимостью
```

## Формат входного файла для универсальной многоэтапной задачи

```json
{
  "problem_type": "finite_horizon",
  "optimization": "min",
  "initial_state": "A",
  "terminal_values": {
    "F": 0,
    "G": 0
  },
  "stages": [
    {
      "name": "1",
      "states": ["A"],
      "transitions": [
        {
          "state": "A",
          "decision": "A-B",
          "next_state": "B",
          "value": 4
        },
        {
          "state": "A",
          "decision": "A-C",
          "next_state": "C",
          "value": 2
        }
      ]
    }
  ]
}
```

Поля:

```text
problem_type     — тип задачи: finite_horizon
optimization     — критерий оптимизации: max или min
initial_state    — начальное состояние
teminal_values   — граничные значения последнего этапа
stages           — список этапов
name             — имя этапа
states           — состояния, доступные на этапе
transitions      — альтернативы перехода
state            — исходное состояние альтернативы
decision         — название решения
next_state       — состояние следующего этапа
value            — локальный выигрыш или локальные затраты
```

## Что записывается в файл снимков

Файл снимков создается в формате Markdown.

Для задачи о загрузке программа записывает:

1. исходную таблицу предметов;
2. таблицу функции Беллмана для каждого этапа;
3. проверенные альтернативы для каждой вместимости;
4. оптимальное количество предметов на каждом этапе;
5. восстановление итогового решения.

Для универсальной многоэтапной задачи программа записывает:

1. граничные значения последнего этапа;
2. таблицу альтернатив для каждого состояния на каждом этапе;
3. оптимальное значение функции Беллмана;
4. выбранное оптимальное решение;
5. восстановленную оптимальную стратегию.

## Используемые рекуррентные соотношения

### Задача о загрузке

Для этапа `i` и оставшейся вместимости `W`:

```text
F_i(W) = opt { r_i * m_i + F_{i+1}(W - w_i * m_i) }
```

где:

```text
m_i = 0, 1, ..., floor(W / w_i)
```

Если для предмета задано `max_count`, то дополнительно выполняется ограничение:

```text
m_i <= max_count
```

### Универсальная многоэтапная задача

Для состояния `s` на этапе `i`:

```text
F_i(s) = opt { c_i(s, a) + F_{i+1}(T_i(s, a)) }
```

где:

```text
a             — альтернатива;
c_i(s, a)     — локальное значение перехода;
T_i(s, a)     — следующее состояние;
opt           — максимум или минимум в зависимости от задачи.
```

## Рекомендации для отчета

2. математическую модель;
3. выделение этапов, состояний и альтернатив;

7. листинг программы.

## Ограничения реализации

Для задачи о загрузке вместимость и веса должны быть целыми числами, потому что таблица динамического программирования строится по дискретным значениям вместимости.

Для универсальной многоэтапной задачи переходы должны быть детерминированными: одна выбранная альтернатива ведет в одно конкретное следующее состояние.
