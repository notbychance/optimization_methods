# Лабораторная работа 6. Детерминированные модели динамического программирования

Программа предназначена для выполнения лабораторной работы по теме **«Детерминированные модели динамического программирования»**.

В методических указаниях для этой лабораторной работы требуется решить математическую модель своего варианта, разработать программу общего вида и составить отчет по работе. Динамическое программирование рассматривается как разбиение многомерной задачи на этапы, где на каждом этапе решается одномерная подзадача, а ключевыми элементами модели являются этапы, состояния и альтернативы.

## Возможности программы

Программа поддерживает три вида детерминированных моделей:

1. `knapsack` — задача о загрузке, то есть задача выбора количества предметов при ограниченной грузоподъемности.
2. `finite_horizon` — универсальная конечная многоэтапная задача с состояниями, альтернативами, переходами и граничными значениями.
3. `multiplicative_constraint_dynamic_programming` — 17-й вариант ЛР 6 из методички.

Алгоритмы и проверки реализованы вручную, без библиотек оптимизации.

## 17-й вариант

Для 17-го варианта задана задача:

```text
Максимизировать: z = y1^2 + y2^2 + ... + yn^2
при условиях:
y1 * y2 * ... * yn = c,
yi >= 0, i = 1, 2, ..., n.
```

В исходном условии параметры `n` и `c` оставлены символическими. Поэтому файлы:

```text
examples/lab6_knapsack_example.json
examples/lab6_finite_horizon_example.json
```

сохраняют условие 17-го варианта как есть. При запуске таких файлов программа завершится штатным статусом:

```text
input_required
```

Это значит, что программа прочитала условие, но не подставила искусственные значения `n` и `c`.

Если задать конкретные `n` и `c`, программа проверит математический статус задачи. Для исходного направления `max` при `n >= 2` задача **не ограничена сверху**, потому что можно построить допустимую последовательность решений, в которой произведение остается равным `c`, а сумма квадратов стремится к бесконечности. Программа не исправляет условие на `min` и не подгоняет его под конечный результат.

Нужен только Python 3.10 или выше.

## Запуск

Для файла 17-го варианта без числовых значений `n` и `c`:

```bash
python main.py --input examples/lab6_knapsack_example.json --snapshot lab6_variant17_snapshot.md
```

Или:

```bash
python main.py --input examples/lab6_finite_horizon_example.json --snapshot lab6_variant17_snapshot.md
```

Для демонстрационного файла с конкретными `n = 3`, `c = 10`:

```bash
python main.py --input examples/lab6_multiplicative_concrete_example.json --snapshot lab6_multiplicative_concrete_snapshot.md
```

Параметры запуска:

```text
--input, -i     путь к JSON-файлу с условием задачи
--snapshot, -s  путь к файлу, куда будут записаны таблицы этапов и выводы
```

## Формат входного файла для 17-го варианта

Вариант без числовых параметров, полностью сохраняющий условие методички:

```json
{
  "source": "МУ_ЛР_Методы_оптимизации, ЛР 6, вариант 17",
  "variant_number": 17,
  "problem_type": "multiplicative_constraint_dynamic_programming",
  "condition_summary": "Решить задачу методом динамического программирования: максимизировать z = y1^2 + y2^2 + ... + yn^2 при условии y1*y2*...*yn = c, yi >= 0, i = 1, 2, ..., n.",
  "optimization": "max",
  "parameters": {
    "n": "n",
    "c": "c"
  }

```

Вариант с конкретными параметрами:

```json
{
  "problem_type": "multiplicative_constraint_dynamic_programming",
  "optimization": "max",
  "parameters": {

    "c": 10
  }

```

## Статусы для 17-го варианта

```text
input_required  — в JSON не заданы конкретные числовые значения n и c
infeasible      — допустимых решений нет, например c < 0 при yi >= 0
unbounded       — задача max не ограничена сверху при n >= 2
optimal         — конечное решение найдено для частного случая n = 1
```

## Что записывается в файл снимков

Файл снимков создается в формате Markdown.

Для 17-го варианта программа записывает:

1. исходное условие;
2. параметры `n` и `c`, прочитанные из JSON;
3. декомпозицию динамического программирования;
4. рекуррентную форму `F_i(p) = sup { y_i^2 + F_{i+1}(p / y_i) }`;
5. проверку необходимости числовых данных;
6. для числового случая — анализ допустимости и ограниченности;
7. итоговый статус задачи.

## Используемые рекуррентные соотношения

### 17-й вариант

```text
F_i(p) = sup { y_i^2 + F_{i+1}(p / y_i) }, y_i > 0
F_n(p) = p^2
```

### Задача о загрузке

```text
F_i(W) = opt { r_i * m_i + F_{i+1}(W - w_i * m_i) }
m_i = 0, 1, ..., floor(W / w_i)
```

### Универсальная многоэтапная задача

```text
F_i(s) = opt { c_i(s, a) + F_{i+1}(T_i(s, a)) }
```
