# ЛР 8. Теория игр и принятия решений

Программа предназначена для выполнения 8-й лабораторной работы по дисциплине **«Методы оптимизации»**.

Тема лабораторной работы: **«Теория игр и принятия решений»**.

Программа реализует вычисления без библиотек оптимизации и без готовых игровых/статистических пакетов. Все алгоритмы написаны вручную на Python.

---

## 1. Возможности программы

Поддерживаются три типа задач:

1. `matrix_game` — матричная игра двух игроков с нулевой суммой.
2. `decision` — задача принятия решений в условиях риска и неопределенности.
3. `bimatrix` — биматричная игра двух игроков с поиском равновесий Нэша в чистых стратегиях.

Для матричной игры программа:

- читает платежную матрицу первого игрока;
- вычисляет нижнюю цену игры;
- вычисляет верхнюю цену игры;
- проверяет наличие седловой точки;
- если седловая точка есть, строит чистые оптимальные стратегии;
- если седловой точки нет, ищет решение в смешанных стратегиях собственным перебором носителей;
- решает системы линейных уравнений собственным методом Гаусса-Жордана;
- выводит цену игры и вероятности стратегий обоих игроков.

Для задачи принятия решений программа рассчитывает:

- критерий Вальда;
- критерий максимакса;
- критерий Лапласа;
- критерий Гурвица;
- критерий Сэвиджа;
- критерий Байеса, если во входном файле заданы вероятности состояний.

Для биматричной игры программа:

- читает две матрицы выигрышей;
- проверяет каждую клетку на взаимный лучший ответ;
- находит равновесия Нэша в чистых стратегиях.

## 4. Запуск

Общий формат запуска:

```bash
python main.py --input путь_к_условию.json --snapshot путь_к_снимкам.md
```

Короткие параметры:

```bash
python main.py -i путь_к_условию.json -s путь_к_снимкам.md
```

Если параметр `--snapshot` не указан, файл снимков будет создан с именем:

```text
game_theory_snapshot.md
```

## 8. Формат входного файла для задачи принятия решений

```json
{
  "task_type": "decision",
  "objective": "max",
  "payoff_matrix": [
    [12, 8, 10, 6],
    [7, 15, 9, 5],
    [10, 9, 11, 10],
    [5, 12, 14, 7]
  ],
  "alternatives": ["A1", "A2", "A3", "A4"],
  "states": ["S1", "S2", "S3", "S4"],
  "probabilities": ["1/4", "1/4", "1/4", "1/4"],
  "hurwicz_alpha": "0.6"
}
```

Поля:

| Поле | Назначение |
|---|---|
| `task_type` | Тип задачи. Для задачи принятия решений — `decision`. |
| `objective` | Направление оптимизации: `max` или `min`. |
| `payoff_matrix` | Матрица выигрышей, затрат или полезностей. |
| `alternatives` | Названия альтернатив. |
| `states` | Названия состояний среды. |
| `probabilities` | Вероятности состояний среды. Нужны для критерия Байеса. Поле необязательное. |
| `hurwicz_alpha` | Коэффициент оптимизма для критерия Гурвица. |

Для задачи минимизации программа преобразует исходную матрицу к матрице полезности путем смены знака элементов. Это позволяет применять те же критерии в единой форме максимизации полезности.

## 14. Возможные статусы результата

| Статус | Значение |
|---|---|
| `optimal_pure` | Для матричной игры найдена седловая точка и чистые оптимальные стратегии. |
| `optimal_mixed` | Для матричной игры найдено решение в смешанных стратегиях. |
| `not_found` | Для матричной игры не удалось найти решение перебором носителей. |
| `calculated` | Для задачи принятия решений или биматричной игры расчеты выполнены. |

---

## 15. Примечания по ограничениям

1. Для матричной игры смешанные стратегии ищутся перебором носителей одинакового размера. Для учебных матриц из лабораторной работы этого достаточно.
2. Для биматричной игры реализован поиск только чистых равновесий Нэша.
3. Внешние библиотеки не используются, поэтому программа подходит для демонстрации собственного алгоритма в лабораторной работе.
