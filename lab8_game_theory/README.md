# Модуль 8: теория игр и принятие решений

Консольная программа для расчёта матричных игр, критериев принятия решений, биматричных игр и дерева решений профилактического обслуживания. Алгоритмы реализованы вручную без игровых, статистических и оптимизационных библиотек.

## Поддерживаемые типы задач

| `task_type` | Назначение |
|---|---|
| `matrix_game` | матричная игра двух игроков с нулевой суммой |
| `decision` | выбор альтернативы по критериям принятия решений |
| `bimatrix` | биматричная игра с поиском равновесий Нэша в чистых стратегиях |
| `preventive_maintenance_decision_tree` | дерево решений для выбора цикла профилактического обслуживания |

## Возможности

Для `matrix_game`:

- вычисление нижней и верхней цены игры;
- поиск седловых точек;
- построение чистых оптимальных стратегий;
- поиск смешанных стратегий перебором носителей;
- решение систем линейных уравнений методом Гаусса-Жордана.

Для `decision`:

- критерий Вальда;
- критерий максимакса;
- критерий Лапласа;
- критерий Гурвица;
- критерий Сэвиджа;
- критерий Байеса при заданных вероятностях состояний.

Для `bimatrix`:

- проверка взаимных лучших ответов;
- поиск равновесий Нэша в чистых стратегиях.

Для `preventive_maintenance_decision_tree`:

- расчёт ожидаемой стоимости цикла обслуживания;
- перебор допустимых длин цикла;
- выбор цикла с минимальной средней стоимостью;
- построение текстового дерева решений в Markdown-протоколе.

## Запуск

Из каталога модуля:

```bash
python main.py -i examples/lab8_example.json -s game_theory_snapshot.md
```

Из корня проекта:

```bash
python main.py -n 8 -i lab8_example.json -o results/lab8.md
```

## Формат JSON для `matrix_game`

```json
{
  "task_type": "matrix_game",
  "matrix": [
    [2, -1],
    [0, 3]
  ],
  "row_names": ["A1", "A2"],
  "column_names": ["B1", "B2"]
}
```

Матрица задаёт выигрыши первого игрока.

## Формат JSON для `decision`

```json
{
  "task_type": "decision",
  "objective": "max",
  "matrix": [
    [10, 4, 6],
    [7, 8, 5],
    [3, 9, 11]
  ],
  "alternative_names": ["A1", "A2", "A3"],
  "state_names": ["S1", "S2", "S3"],
  "probabilities": [0.2, 0.5, 0.3],
  "hurwicz_alpha": 0.6
}
```

Поля `probabilities` и `hurwicz_alpha` необязательны. Если вероятности не заданы, критерий Байеса не рассчитывается.

## Формат JSON для `bimatrix`

```json
{
  "task_type": "bimatrix",
  "first_player_matrix": [
    [3, 1],
    [0, 2]
  ],
  "second_player_matrix": [
    [2, 0],
    [1, 3]
  ],
  "row_names": ["A1", "A2"],
  "column_names": ["B1", "B2"]
}
```

Обе матрицы должны иметь одинаковый размер.

## Формат JSON для `preventive_maintenance_decision_tree`

```json
{
  "task_type": "preventive_maintenance_decision_tree",
  "fleet_size": 20,
  "failure_probability_by_month": {
    "1": 0.0,
    "2": 0.03,
    "3": 0.04,
    "11_and_later": 0.13
  },
  "random_failure_cost": 200,
  "preventive_repair_cost": 75,
  "max_cycle_months": 12
}
```

Поля:

| Поле | Описание |
|---|---|
| `fleet_size` | количество единиц техники |
| `failure_probability_by_month` | вероятность отказа по месяцам эксплуатации |
| `random_failure_cost` | стоимость случайного отказа одной единицы |
| `preventive_repair_cost` | стоимость планового обслуживания одной единицы |
| `max_cycle_months` | максимальная длина цикла перебора; необязательно |

Если указан ключ `11_and_later`, его вероятность используется для всех месяцев после 10-го, когда конкретное значение месяца отсутствует.

## Результат

В консоль выводятся статус, сообщение и итоговые значения. Markdown-файл содержит исходные матрицы, промежуточные таблицы, расчёты критериев, равновесия, стратегии или дерево решений.

## Возможные статусы

| Статус | Смысл |
|---|---|
| `optimal_pure` | матричная игра решена в чистых стратегиях |
| `optimal_mixed` | матричная игра решена в смешанных стратегиях |
| `not_found` | смешанное решение не найдено в перебранных носителях |
| `calculated` | расчёт критериев или биматричной игры выполнен |
| `optimal_cycle_found` | найден оптимальный цикл обслуживания |
