# Лабораторная работа №4 — Решение транспортных задач

Программа решает транспортную задачу минимизации суммарных транспортных расходов.
Реализация написана на Python без специализированных библиотек оптимизации.

## Назначение

Программа предназначена для выполнения лабораторной работы «Решение транспортных задач» по дисциплине «Методы оптимизации».

Поддерживается общий вид транспортной задачи:

\[
F = \sum_i \sum_j c_{ij}x_{ij} \rightarrow \min
\]

при ограничениях:

\[
\sum_j x_{ij} = a_i,
\]

\[
\sum_i x_{ij} = b_j,
\]

\[
x_{ij} \ge 0.
\]

где:

- `c_ij` — тариф перевозки единицы груза от поставщика `i` к потребителю `j`;
- `x_ij` — количество груза, перевозимое по маршруту `i -> j`;
- `a_i` — запас поставщика;
- `b_j` — потребность потребителя.

## Возможности программы

Программа включает четыре основных модуля:

1. `transportation.py` — собственный алгоритм решения транспортной задачи:
   - балансировка задачи;
   - построение начального опорного плана;
   - метод северо-западного угла;
   - метод наименьшей стоимости;
   - метод Фогеля;
   - метод потенциалов для улучшения плана;
   - построение цикла перераспределения;
   - обработка вырожденного плана через нулевые базисные клетки.
2. `snapshot.py` — поэтапный вывод таблиц в Markdown-файл:
   - исходная сбалансированная транспортная таблица;
   - начальный опорный план;
   - потенциалы `u_i` и `v_j`;
   - оценки свободных клеток `delta_ij`;
   - цикл перераспределения;
   - итоговый оптимальный план.
3. `problem_reader.py` — чтение условия задачи из JSON-файла.
4. `main.py` — модуль запуска из командной строки.

## Требования

Достаточно установленного Python версии 3.10 или выше.

Внешние библиотеки не требуются.

## Запуск

Перейдите в папку проекта:

```bash
cd lab4_transportation_python
```

Запустите пример:

```bash
python main.py --input examples/lab4_transport_example.json --snapshot transportation_snapshot.md
```

Сокращенная форма запуска:

```bash
python main.py -i examples/lab4_transport_example.json -s transportation_snapshot.md
```

После запуска в консоль выводятся:

- статус решения;
- количество итераций метода потенциалов;
- минимальная стоимость перевозок;
- итоговая матрица перевозок;
- путь к файлу со снимками алгоритма.

## Формат входного файла

Входной файл должен иметь формат JSON.

Пример:

```json
{
  "initial_method": "vogel",
  "source_names": ["A1", "A2", "A3"],
  "destination_names": ["B1", "B2", "B3", "B4"],
  "supply": [20, 30, 25],
  "demand": [10, 25, 25, 15],
  "costs": [
    [8, 6, 10, 9],
    [9, 7, 4, 2],
    [3, 4, 2, 5]
  ]
}
```

Описание полей:

| Поле | Назначение |
|---|---|
| `initial_method` | Метод построения начального опорного плана. Возможные значения: `north_west`, `least_cost`, `vogel`. |
| `source_names` | Имена поставщиков. Поле необязательное. |
| `destination_names` | Имена потребителей. Поле необязательное. |
| `supply` | Запасы поставщиков. |
| `demand` | Потребности потребителей. |
| `costs` | Матрица тарифов перевозки. |

Количество строк в `costs` должно совпадать с количеством элементов `supply`.
Количество столбцов в `costs` должно совпадать с количеством элементов `demand`.

## Методы построения начального плана

В поле `initial_method` можно указать один из трех методов.

### Метод северо-западного угла

```json
"initial_method": "north_west"
```

Начальное заполнение начинается с левой верхней клетки таблицы.
Метод прост в вычислениях, но часто дает не самый хороший начальный план.

### Метод наименьшей стоимости

```json
"initial_method": "least_cost"
```

На каждом шаге выбирается доступная клетка с минимальным тарифом.
Метод обычно дает более выгодный начальный план, чем метод северо-западного угла.

### Метод Фогеля

```json
"initial_method": "vogel"
```

На каждом шаге рассчитываются штрафы по строкам и столбцам, затем выбирается направление с максимальным штрафом.
Часто дает хороший начальный опорный план.

## Балансировка задачи

Если суммарный запас не равен суммарной потребности, программа автоматически приводит транспортную задачу к закрытому виду.

Если запас больше потребности, добавляется фиктивный потребитель с нулевыми тарифами.

Если потребность больше запаса, добавляется фиктивный поставщик с нулевыми тарифами.

Пример несбалансированной задачи находится в файле:

```text
examples/lab4_unbalanced_example.json
```

Запуск:

```bash
python main.py --input examples/lab4_unbalanced_example.json --snapshot unbalanced_snapshot.md
```

## Файл снимков алгоритма

Файл снимков создается в формате Markdown.

В нем фиксируются:

1. исходная сбалансированная транспортная задача;
2. начальный опорный план;
3. текущая стоимость плана;
4. базисные клетки;
5. потенциалы `u_i` и `v_j`;
6. оценки свободных клеток `delta_ij = c_ij - u_i - v_j`;
7. выбранная свободная клетка;
8. цикл перераспределения;
9. значение `theta`;
10. план после перераспределения;
11. итоговый оптимальный план.

В транспортной таблице ячейка записывается в формате:

```text
перевозка / тариф
```

Звездочка `*` означает, что клетка является базисной.

Пример:

```text
20 / 6*
```

Это означает, что по маршруту перевозится 20 единиц груза, тариф равен 6, клетка входит в базис.

## Логика метода потенциалов

После построения начального плана программа выполняет следующие шаги:

1. Для базисных клеток строится система потенциалов:

   ```text
   u_i + v_j = c_ij
   ```

2. Для каждой свободной клетки рассчитывается оценка:

   ```text
   delta_ij = c_ij - u_i - v_j
   ```

3. Если все оценки свободных клеток неотрицательны, план оптимален.

4. Если есть отрицательные оценки, выбирается свободная клетка с наименьшей оценкой.

5. Для выбранной клетки строится замкнутый цикл перераспределения.

6. В вершинах цикла попеременно ставятся знаки `+` и `-`.

7. Значение `theta` выбирается как минимальная перевозка в отрицательных вершинах цикла.

8. Значение `theta` прибавляется в положительных вершинах и вычитается в отрицательных.

9. Одна из нулевых отрицательных клеток выводится из базиса.

10. Проверка оптимальности повторяется.

Для файла `examples/lab4_transport_example.json` программа выводит:

```text
Статус: optimal
Сообщение: Оптимальный транспортный план найден методом потенциалов.
Количество итераций: 0
Минимальная стоимость перевозок: 280
План перевозок:
  Поставщик/потребитель | B1 | B2 | B3 | B4
  A1                    | 0  | 20 | 0  | 0
  A2                    | 0  | 0  | 15 | 15
  A3                    | 10 | 5  | 10 | 0
```

Количество итераций может отличаться при выборе другого метода построения начального плана.
Например, при `north_west` для того же примера программа выполняет дополнительные улучшения плана методом потенциалов.

## Ограничения

Программа решает классическую транспортную задачу на минимум.
Для задач на максимум прибыль можно преобразовать в задачу на минимум, заменив тарифы на потери или вычитая прибыли из достаточно большой константы.
