# Модуль 4: транспортная задача

Консольная программа для решения транспортной задачи. Модуль строит начальный допустимый план одним из поддерживаемых методов, затем оптимизирует его методом потенциалов и сохраняет все промежуточные таблицы в Markdown.

## Возможности

- закрытые и открытые транспортные задачи;
- автоматическое добавление фиктивного поставщика или потребителя для балансировки;
- методы начального плана: `north_west`, `least_cost`, `vogel`;
- оптимизация методом потенциалов;
- построение циклов перераспределения;
- точные вычисления через `fractions.Fraction`;
- Markdown-снимки начального плана, потенциалов, оценок и улучшений;
- статус `optimal` при найденном оптимальном плане.

## Запуск

Из каталога модуля:

```bash
python main.py -i examples/lab4_example.json -s transportation_snapshot.md
```

Из корня проекта:

```bash
python main.py -n 4 -i lab4_example.json -o results/lab4.md
```

## Формат JSON

```json
{
  "initial_method": "vogel",
  "source_names": ["A1", "A2"],
  "destination_names": ["B1", "B2", "B3"],
  "supply": [100, 150],
  "demand": [80, 90, 80],
  "costs": [
    [4, 6, 8],
    [5, 3, 7]
  ]
}
```

Поля:

| Поле | Описание |
|---|---|
| `initial_method` | `north_west`, `least_cost` или `vogel`; если поле отсутствует, используется метод по умолчанию |
| `source_names` | имена поставщиков; необязательно |
| `destination_names` | имена потребителей; необязательно |
| `supply` | запасы поставщиков |
| `demand` | потребности потребителей |
| `costs` | матрица тарифов перевозки |

Размер `costs` должен соответствовать количеству поставщиков и потребителей.

## Результат

В консоль выводятся статус, сообщение, число итераций, минимальная стоимость и план перевозок. Markdown-файл содержит исходную задачу, начальный план, потенциалы, оценки свободных клеток, циклы перераспределения и итоговый план.
